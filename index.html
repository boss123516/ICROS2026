Abstract

This project proposes an LLM-based framework for decentralized multi-robot cooperation. Instead of designing a separate controller for every robot, every robot executes the same local-level Behavior Tree, called an Identical BT, and chooses what to do from its own local observations.

The Identical BT is composed of a Contribution Check, an auction-based Assign Check, and an LLM-synthesized Problem Solving BT. Instance-specific subtrees are first generated from sampled global plans, then generalized into a reusable local action subtree through inductive synthesis.

Method: Identical BT for Local Cooperation

The main idea is simple: give every robot the same decision tree, but let each robot read the tree through its own local situation. Like workers following the same rulebook in different positions, robots can make different choices without needing different controllers.

1. Contribution Check

Determines whether the robot can contribute to the current task from its local observation. Robots that cannot help simply wait.

2. Assign Check

Uses an auction/MRTA-style check to prevent multiple robots from trying to perform the same task at the same time.

3. Problem Solving BT

Executes the assigned local action using the LLM-synthesized subtree generalized from instance-specific demonstrations.

Experiments

Experiments are conducted in a MuJoCo-based grid simulator with fixed ROBOTIS OMX-F robots. Each robot can pick and place cubes only inside its neighboring 2×2 grid, so cooperation is required to relay cubes to their goal cells.

Generalization

To evaluate scalability, we tested the same synthesized BT — without any modification — on a 7×7 grid with 16 cubes, and on 5×5 grids with added environmental disturbances. A dummy cube is a cube with no assigned goal. A forbidden zone is a cell that cubes cannot pass through. A dead robot is a robot that stops functioning.
